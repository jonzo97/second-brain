## The big new direction: Open Brain

**Source:** YouTube video by Nate — "Open Brain" concept (https://youtu.be/2JiMmye2ezg)

**Core idea:** Replace markdown-file-based second brain with a database-backed, agent-readable knowledge system accessible via MCP from any AI tool. Build for the agent web, not the human web.

**His architecture:**
- PostgreSQL + pgvector on Supabase (free tier)
- Edge function for capture: generates vector embedding + LLM extracts metadata (people, topics, type, action items)
- MCP server for retrieval: semantic search, list recent, stats
- Capture from anywhere: Slack, Claude, ChatGPT, Cursor — any MCP-compatible client
- Cost: ~$0.10-0.30/month

**Why it resonates with our setup:**
- Our topic files (agents.md, rag.md, etc.) are rich but not semantically searchable
- Cross-project mailbox is linear scan only
- Research switchboard is a markdown table, not a queryable system
- We already have CozoDB recipes (graphrag, episodic memory, md ingestion) — graph-native alternative to pgvector
- We've been feeling the friction of "lots of curated knowledge, hard to surface it programmatically"
